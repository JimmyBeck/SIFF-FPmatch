SIFF 排班助手 — 技术说明文档

版本：v1.0.0 · 项目：SIFF260520
最后更新：2026-05-23 · 配套 操作手册

一、系统概述

SIFF 排班助手是面向上海国际电影节字幕组排班工作的 Web 应用。系统将两类 Excel 源数据——电影排片表与字幕员时间表——整合为可编辑、可自动分配、可导出的排班总表，实现「每一场电影 → 两名字幕员」的完整排班闭环。

1.1 标准模板文件

系统内置两份标准 Excel 模板，存放于 public/templates/：

按钮模板文件用途
下载字幕员模板后五天问卷调查 给静静.xlsx字幕员姓名、负责区域、搭档、各日各时段可用性
下载排片表模板2024排片表.xlsx电影节各影院、影厅、时间段的排片信息

下载实现：src/lib/downloadTemplate.ts，通过 fetch 获取静态文件并触发浏览器下载。

1.2 技术栈

React 18 · TypeScript · Vite 7 · React Router 7 · Tailwind CSS · SheetJS (xlsx) · react-dnd · Framer Motion · Zod · localStorage 持久化

二、系统架构

2.1 路由

路径页面功能
/仪表盘数据导入、模板下载、流程导航
/schedule-table电影排片只读查看排片表
/subtitler字幕员表只读查看字幕员可用性
/schedule排班编辑日历/矩阵视图分配字幕员
/schedule-overview排班总表主编辑页、自动排班、导出
/settings设置行政区优先级、排班约束

2.2 三层数据 Context

ScheduleTableContext → ParsedScheduleTable（排片原始数据） ScheduleContext → ScheduleTable（排班总表：cells + assignments） SubtitlerContext → ParsedSchedule（字幕员原始数据）

Provider 嵌套：SubtitlerProvider → ScheduleTableProvider → ScheduleProvider

三、核心数据模型

3.1 排片表 ParsedScheduleTable

  • 解析：src/lib/parseScheduleTable.ts
  • 字段：dates, cinemas, halls, timeSlots, rows[]
  • 影片时间段：08:30, 09:30, 10:00, 10:30, 13:00, 15:30, 18:00, 18:30, 20:40
  • localStorage：schedule_table_data_v2

3.2 字幕员表 ParsedSchedule

  • 解析：src/lib/parseSubtitlerExcel.ts
  • 字段：rows[]（name, schedule, districts, partner）
  • 字幕员时间段：9:00, 10:30, 13:00, 14:00, 15:00, 17:00, 18:30, 20:40, 23:00
  • 格子语义：null = 空闲（可分配），有值 = 已占用
  • localStorage：subtitler_data_v2

3.3 排班总表 ScheduleTable

interface ScheduleTable {
  cells: ScheduleCell[];
  assignments: { [key: string]: ScheduleAssignment };
}
// key = `${date}|${cinema}|${hall}|${timeSlot}`
  • 构建：buildScheduleTableFromParsedData()
  • 合并：syncScheduleTableWithParsedData()
  • localStorage：schedule_table_assignments_v2

3.4 时间映射 TIME_MAPPING

影片时间与字幕员时间通过 src/lib/scheduleConstants.ts 中的 TIME_MAPPING 对应。日期格式：排片表 2024-06-18 ↔ 字幕员表 18日。

3.5 影院行政区映射

CINEMA_DISTRICTS 维护影院到上海 11 个行政区的映射，用于自动排班区域过滤。

四、数据同步机制

触发行为
上传排片表setScheduleData → useScheduleTableSync → syncFromParsedData 合并 cells
上传字幕员表setSubtitlerData，不影响已有 assignments
updateAssignment更新 assignments 并写入 localStorage
排班总表页 scheduleData 变化重建 cells（buildScheduleTableFromParsedData）

五、各页面与排班总表的关系

页面读总表写总表说明
仪表盘否间接上传排片表重建 cells；模板下载不影响总表
电影排片否否只读参照，有电影的格子 = 总表有 cell
字幕员表否否约束数据源，决定下拉可选人
排班编辑是是日历/矩阵写入 assignments
排班总表是是主编辑 + 自动排班 + 导出 Excel
设置否间接约束影响自动排班结果

六、自动排班算法

入口：排班总表 → 自动排班模式 → 执行自动排班
实现：src/lib/autoSchedule.ts

  1. 读取 settings_constraints 与 settings_districts
  2. 保留已有手动分配（manualAssignments）
  3. 按行政区优先级逐区处理
  4. 每场：匹配区域 → 过滤空闲字幕员 → 检查 daily/total/互斥 → 优先搭档 → 写入 subtitler1/2

默认约束：2 场/天、5 场/总计、时段互斥、优先时间块少的字幕员。

七、导出

排班总表页「导出 Excel」生成 排班总表_YYYYMMDD.xlsx：按日期分 Sheet；列含影院、影厅、日期、星期、时间、电影、字幕员1、字幕员2；未识别区域影院行标灰。

八、localStorage 键名

SIFF 排班助手 — 截图版操作手册

版本：v1.0.0 · 2026-05-23
配套技术文档：技术说明

本手册按实际界面截图，逐步说明从下载模板到导出排班总表的完整操作流程。

1. 快速开始

推荐流程：

下载模板 → 填写 Excel → 上传两份数据 → 设置规则 → 自动/手动排班 → 导出 Excel

左侧导航栏六个入口：

菜单作用
仪表盘上传数据、下载模板
排班编辑全局日历/矩阵分配
字幕员表查看人员可用性
电影排片查看排片结构
排班总表主编辑 + 导出
设置行政区优先级、排班约束

2. Step 1：下载并填写标准模板

打开 仪表盘，在「数据导入」卡片底部有两个下载按钮。

2.1 下载字幕员模板

  1. 点击金色按钮 「下载字幕员模板」
  2. 浏览器将下载：后五天问卷调查 给静静.xlsx
  3. 用 Excel 填写或核对：姓名、区域（负责哪个区填 1）、搭档、各日期-时间段列（空白 = 有空，有内容 = 已占用）

2.2 下载排片表模板

  1. 点击灰色按钮 「下载排片表模板」
  2. 浏览器将下载：2024排片表.xlsx
  3. 填写日期、周、影院、影厅及各时间段（08:30 ~ 20:40）的电影信息

提示：模板文件与系统解析逻辑完全匹配，建议直接基于模板修改，避免表头格式不兼容。

3. Step 2：上传数据

仍在仪表盘页面：

3.1 上传字幕员数据

  1. 点击 「字幕员数据」 标签（默认选中）
  2. 将填好的 Excel 拖入虚线框，或点击选择文件
  3. 解析成功后，底部状态栏显示绿色 ✓ 及文件名
  4. 弹出提示时可选择跳转到「字幕员表」查看

3.2 上传电影排片表

  1. 点击 「电影排片表」 标签
  2. 上传 2024排片表.xlsx
  3. 解析成功后，底部显示文件名，可选择跳转到「电影排片」查看

3.3 对排班总表的影响

上传内容总表变化
排片表重建所有 cells（场次行），已有分配尽量保留
字幕员表不改变总表结构，仅影响后续可选字幕员

4. Step 3：核对电影排片

进入 电影排片 页面：

区域操作
顶部日期标签点击切换日期（如 6月14日 五）
左侧影院列表点击选择影院
右侧表格查看 影厅 × 时间段 的电影信息
底部统计查看影院数、场次数
上传/更新数据返回仪表盘重新上传

与总表关系：表格中有绿色内容的格子 = 排班总表中对应场次需要分配 2 名字幕员。本页只读，不能直接改排班。

5. Step 4：核对字幕员可用性

进入 字幕员表 页面：

区域操作
顶部日期标签切换 18日、19日… 查看不同日期
区域列绿色「1」= 负责该区域
时间段列绿色 = 有空（可分配），有字 = 已占用
搭档列固定搭档，自动排班时优先配对

与总表关系：总表分配时的下拉列表，只显示此处为「绿色空闲」的字幕员。若下拉为空，先回到此页确认该时段是否有人有空。

6. Step 5：配置排班规则（可选）

进入 设置 页面：

6.1 行政区优先级

  • 拖动列表调整 11 个区的顺序
  • 越靠前，自动排班时越先处理
  • 点击 保存设置 生效

6.2 排班大前提条件

设置项默认值说明
每天排班限制2 场/天自动排班生效
总场次限制5 场/总计自动排班生效
同一时间段互斥开启同一人同时只能去一个厅
优先时间块多的字幕员开启优先安排剩余时段少的人

与总表关系：仅影响 自动排班 的分配结果，手动下拉不受这些上限强制拦截。

7. Step 6：排班编辑（日历/矩阵）

进入 排班编辑 页面：

7.1 工具栏

按钮功能
切换至矩阵视图日历 ↔ 表格两种布局
手动排班启用下拉编辑
只读模式禁止修改，仅查看
进度统计共 N 场 · 已完成 M 场

7.2 日历视图

  • 左侧：字幕员列表，可 拖拽 到右侧场次 → 快速填入字幕员1
  • 右侧：按日期分组的场次卡片
  • 每场卡片有两个下拉：字幕员1（蓝）、字幕员2（绿）
  • 双人填完 → 卡片变绿；冲突 → 变红

7.3 矩阵视图

表格列：日期/时间 | 影片 | 字幕员1 | 字幕员2，适合逐行批量操作。

与总表关系：此处每次选择/拖拽都会 实时写入排班总表（assignments），与「排班总表」页完全同步。

8. Step 7：排班总表（主工作区）

进入 排班总表 页面 — 这是最终核对与导出的主界面：

8.1 头部操作

控件说明
手动排班表格内出现可编辑下拉
自动排班切换为只读 + 显示「执行自动排班」按钮
清除清空所有已分配字幕员（需确认）
导出 Excel下载完整排班结果
执行自动排班按规则批量填充（保留已有手动分配）

8.2 主体布局

与「电影排片」类似，但表格多了两列：

列可编辑写入总表字段
影厅 / 时间 / 电影否—
字幕员1是（手动模式）assignments[key].subtitler1
字幕员2是（手动模式）assignments[key].subtitler2

8.3 推荐操作顺序

  1. 切换 自动排班 → 点击 执行自动排班
  2. 查看头部进度「已分配 X / Y 场」
  3. 切换 手动排班 → 逐影院补全未分配场次
  4. 全部完成后 → 导出 Excel

9. Step 8：导出结果

在排班总表页点击 「导出 Excel」：

  • 文件名：排班总表_YYYYMMDD.xlsx
  • 每个 festival 日期一个 Sheet
  • 列：影院 | 影厅 | 日期 | 星期 | 时间 | 电影 | 字幕员1 | 字幕员2
  • 未识别区域的影院行以灰色标记

导出文件即为可交付的最终排班总表。

10. 常见问题

Q：模板下载没反应？
  • 确认使用 Chrome / Edge / Safari 等现代浏览器
  • 检查是否被浏览器拦截下载
  • 开发环境需确保 public/templates/ 下两个 xlsx 文件存在
Q：上传后排班总表是空的？
需先上传 电影排片表，系统才能生成 cells；仅上传字幕员表不会产生场次。
Q：下拉列表显示「此时段无空闲字幕员」？
  1. 到字幕员表确认该日期、时段是否有绿色格子
  2. 确认是否已被同场另一字幕员选中
  3. 确认是否已上传字幕员数据
Q：排班编辑和排班总表有什么区别？
  • 排班编辑：全局浏览，支持拖拽，适合宏观分配
  • 排班总表：按影院逐日核对，支持自动排班和导出
  • 两者操作同一份数据，互通同步
Q：刷新页面数据会丢吗？
不会。数据保存在浏览器 localStorage。清除浏览器缓存会导致丢失。
