## 一、系统概述

**SIFF 排班助手**是面向上海国际电影节字幕组排班工作的 Web 应用。系统将两类 Excel 源数据——**电影排片表**与**字幕员时间表**——整合为可编辑、可自动分配、可导出的**排班总表**，实现「每一场电影 → 两名字幕员」的完整排班闭环。

### 1.1 标准模板文件

系统内置两份标准 Excel 模板，存放于 `public/templates/`：

| 按钮 | 模板文件 | 用途 |
|------|----------|------|
| 下载字幕员模板 | `后五天问卷调查 给静静.xlsx` | 字幕员姓名、负责区域、搭档、各日各时段可用性 |
| 下载排片表模板 | `2024排片表.xlsx` | 电影节各影院、影厅、时间段的排片信息 |

下载实现：`src/lib/downloadTemplate.ts`，通过 `fetch` 获取静态文件并触发浏览器下载。原始文件同时保留在 `assets/` 目录作为源文件。

### 1.2 技术栈

React 18 · TypeScript · Vite 7 · React Router 7 · Tailwind CSS · SheetJS (xlsx) · react-dnd · Framer Motion · Zod · localStorage 持久化

## 二、系统架构

### 2.1 路由

| 路径 | 页面 | 功能 |
|------|------|------|
| `/` | 仪表盘 | 数据导入、模板下载、流程导航 |
| `/schedule-table` | 电影排片 | 只读查看排片表 |
| `/subtitler` | 字幕员表 | 只读查看字幕员可用性 |
| `/schedule` | 排班编辑 | 日历/矩阵视图分配字幕员 |
| `/schedule-overview` | 排班总表 | 主编辑页、自动排班、导出 |
| `/settings` | 设置 | 行政区优先级、排班约束 |

### 2.2 三层数据 Context

```
ScheduleTableContext  →  ParsedScheduleTable（排片原始数据）
ScheduleContext       →  ScheduleTable（排班总表：cells + assignments）
SubtitlerContext      →  ParsedSchedule（字幕员原始数据）
```

Provider 嵌套：`SubtitlerProvider` → `ScheduleTableProvider` → `ScheduleProvider`

## 三、核心数据模型

### 3.1 排片表 ParsedScheduleTable

- **解析**：`src/lib/parseScheduleTable.ts`
- **字段**：dates, cinemas, halls, timeSlots, rows[]
- **影片时间段**：08:30, 09:30, 10:00, 10:30, 13:00, 15:30, 18:00, 18:30, 20:40
- **localStorage**：`schedule_table_data_v2`

### 3.2 字幕员表 ParsedSchedule

- **解析**：`src/lib/parseSubtitlerExcel.ts`
- **字段**：rows[]（name, schedule, districts, partner）
- **字幕员时间段**：9:00, 10:30, 13:00, 14:00, 15:00, 17:00, 18:30, 20:40, 23:00
- **格子语义**：`null` = 空闲（可分配），有值 = 已占用
- **localStorage**：`subtitler_data_v2`

### 3.3 排班总表 ScheduleTable

```typescript
interface ScheduleTable {
  cells: ScheduleCell[];
  assignments: { [key: string]: ScheduleAssignment };
}
// key = `${date}|${cinema}|${hall}|${timeSlot}`
```

- **构建**：`buildScheduleTableFromParsedData()` — 从排片表生成 cells
- **合并**：`syncScheduleTableWithParsedData()` — 更新排片时保留已有 assignments
- **localStorage**：`schedule_table_assignments_v2`

### 3.4 时间映射 TIME_MAPPING

影片时间与字幕员时间通过 `src/lib/scheduleConstants.ts` 中的 `TIME_MAPPING` 对应。日期格式：排片表 `2024-06-18` ↔ 字幕员表 `18日`。

### 3.5 影院行政区映射

`CINEMA_DISTRICTS` 维护影院到上海 11 个行政区的映射，用于自动排班区域过滤。

## 四、数据同步机制

| 触发 | 行为 |
|------|------|
| 上传排片表 | `setScheduleData` → `useScheduleTableSync` → `syncFromParsedData` 合并 cells |
| 上传字幕员表 | `setSubtitlerData`，不影响已有 assignments |
| `updateAssignment` | 更新 assignments 并写入 localStorage |
| 排班总表页 scheduleData 变化 | 重建 cells（`buildScheduleTableFromParsedData`） |

## 五、各页面与排班总表的关系

| 页面 | 读总表 | 写总表 | 说明 |
|------|--------|--------|------|
| 仪表盘 | 否 | 间接 | 上传排片表重建 cells；模板下载不影响总表 |
| 电影排片 | 否 | 否 | 只读参照，有电影的格子 = 总表有 cell |
| 字幕员表 | 否 | 否 | 约束数据源，决定下拉可选人 |
| 排班编辑 | 是 | 是 | 日历/矩阵写入 assignments |
| 排班总表 | 是 | 是 | 主编辑 + 自动排班 + 导出 Excel |
| 设置 | 否 | 间接 | 约束影响自动排班结果 |

## 六、自动排班算法

**入口**：排班总表 → 自动排班模式 → 执行自动排班  
**实现**：`src/lib/autoSchedule.ts`

流程概要：
1. 读取 `settings_constraints` 与 `settings_districts`
2. 保留已有手动分配（manualAssignments）
3. 按行政区优先级逐区处理
4. 每场：匹配区域 → 过滤空闲字幕员 → 检查 daily/total/互斥 → 优先搭档 → 写入 subtitler1/2

默认约束：2 场/天、5 场/总计、时段互斥、优先时间块少的字幕员。

## 七、导出

排班总表页「导出 Excel」生成 `排班总表_YYYYMMDD.xlsx`：
- 按日期分 Sheet
- 列：影院 | 影厅 | 日期 | 星期 | 时间 | 电影 | 字幕员1 | 字幕员2
- 未识别区域影院行标灰

## 十、模板下载技术实现

```typescript
// src/lib/downloadTemplate.ts
export const TEMPLATE_FILES = {
  subtitler: {
    path: '/templates/后五天问卷调查 给静静.xlsx',
    filename: '后五天问卷调查 给静静.xlsx',
  },
  schedule: {
    path: '/templates/2024排片表.xlsx',
    filename: '2024排片表.xlsx',
  },
};
```

Vite 将 `public/templates/` 下的文件原样发布到站点根路径 `/templates/...`，构建产物 `dist/templates/` 同样可用。
